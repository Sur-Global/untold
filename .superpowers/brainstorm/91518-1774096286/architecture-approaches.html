Architecture Approaches for UNTOLD.ink

Choose the foundation for the platform

A

Next.js App Router + Supabase — Recommended

Next.js 14 App Router (SSR + SSG) · Supabase for auth, DB, storage, realtime · Supabase Auth with RLS for role-based access · Tailwind + shadcn/ui (matching Figma) · DeepL or OpenAI for auto-translation · Vercel hosting

Pros

  • One coherent stack — auth, DB, storage, realtime in one place
  • RLS enforces role permissions at DB level (secure by default)
  • Matches Figma Make component library exactly (shadcn/ui)
  • SSR for SEO on articles, SSG for static pages
  • Lowest operational complexity

Cons

  • Admin/editorial CMS is custom-built (not a plug-in)
  • Media handling (video/podcast) needs Supabase Storage + CDN config
B

Next.js + Supabase + Sanity CMS

Same stack as A but adds Sanity as a headless CMS for content editing · Editors get a polished editorial UI (like Medium's editor) for free · Supabase still handles auth, users, likes/bookmarks

Pros

  • Best editorial writing experience — rich text, drag-drop, image upload
  • Non-technical authors can publish without touching code
  • Structured content schema maps cleanly to content types

Cons

  • Two backends to manage (Sanity + Supabase)
  • Extra cost ($99/mo+ for team plan)
  • Adds complexity to the translation pipeline
C

Next.js + Supabase + Tiptap Editor

Approach A but with Tiptap (open source) as the rich text editor for article authoring · Full control over the writing experience · Powers the built-in CMS from the Figma design

Pros

  • Custom editor that matches UNTOLD's brand — no generic admin UI
  • Open source, no extra cost
  • Stores content as structured JSON in Supabase — easy to translate
  • Can build exactly the CMS shown in Figma (ContentManagementSystem.tsx)

Cons

  • More frontend work to build the editor experience
  • Collaborative editing not included (Tiptap Pro costs extra)

My Recommendation

Option C — Next.js + Supabase + Tiptap. The Figma already has a fully designed CMS editor (ContentManagementSystem.tsx, VisualEditor.tsx). Building it with Tiptap keeps everything in one stack, gives authors a great writing experience, and stores content as structured JSON that's easy to auto-translate. No extra cost, no second backend.

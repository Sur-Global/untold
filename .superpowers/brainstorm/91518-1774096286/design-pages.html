Design: Pages & Routing

Section 3 of 5 — Next.js App Router structure

Route Structure

Public Routes (SSR)

  • /[locale] — homepage (curated feed)
  • /[locale]/articles — all articles
  • /[locale]/articles/[slug] — article detail
  • /[locale]/videos/[slug] — video detail
  • /[locale]/podcasts/[slug] — podcast detail
  • /[locale]/pills/[slug] — pill detail
  • /[locale]/courses/[slug] — course detail
  • /[locale]/tag/[slug] — tag page
  • /[locale]/author/[slug] — author profile
  • /[locale]/user/[slug] — user profile
  • /[locale]/search — search results
  • /[locale]/about
  • /[locale]/contact
  • /[locale]/newsletter
  • /[locale]/become-creator

Auth-gated Routes

  • /[locale]/dashboard — user home
  • /[locale]/dashboard/bookmarks
  • /[locale]/dashboard/settings
  • /[locale]/create — pick content type
  • /[locale]/create/article
  • /[locale]/create/video
  • /[locale]/create/podcast
  • /[locale]/create/pill
  • /[locale]/edit/[id] — edit any content

Admin-only Routes

  • /[locale]/admin — dashboard
  • /[locale]/admin/content — CMS
  • /[locale]/admin/users — user mgmt
  • /[locale]/admin/moderation
  • /[locale]/admin/analytics
  • /[locale]/create/course — admin only

Locale Routing

Supported locales: es (default) and en — handled by next-intl.

Language switcher in nav changes locale prefix. Article content served from content_translations for the active locale. Falls back to source language if translation pending.

SEO: hreflang tags generated automatically per page.

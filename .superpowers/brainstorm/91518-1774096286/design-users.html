Design: User System & Permissions

Section 4 of 5 — roles, content visibility, auth flow

Three Roles

👑 Admin

  • ✅ All content types incl. courses
  • ✅ Content appears on homepage
  • ✅ Manage users & roles
  • ✅ Moderation panel
  • ✅ Analytics dashboard
  • ✅ Platform settings
  • Profile at /author/[slug]

✍️ Author

  • ✅ Articles, videos, podcasts, pills
  • ✅ Content appears on homepage
  • ✅ Priority in search results
  • ✅ Author badge on profile
  • ❌ Cannot create courses
  • ❌ No admin panel access
  • Profile at /author/[slug]

👤 User

  • ✅ Articles, videos, podcasts, pills
  • ✅ Like & bookmark content
  • ✅ Public profile page
  • ❌ Content NOT on homepage
  • ❌ Lower search priority
  • ❌ Cannot create courses
  • Profile at /user/[slug]

Auth Flow

Sign up
  1. Email + password via Supabase Auth
  2. Email confirmation required
  3. Profile created with role = user
  4. Redirected to complete profile (name, slug)
Role promotion
  1. User applies via "Become a Creator" page
  2. Admin reviews & promotes to author in admin panel
  3. RLS enforced at DB level — role column only writable by admin

Homepage Feed Logic

A Postgres view (homepage_feed) filters: status = published AND author.role IN ('admin','author'). Ordered by published_at DESC, with is_featured items pinned first. Search returns all content but ranks admin/author results above user content.

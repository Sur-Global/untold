Design: Data Model

Section 2 of 5 — Supabase Postgres schema

Core Tables

profiles

  • id uuid (→ auth.users)
  • slug text unique
  • display_name text
  • role enum: admin|author|user
  • bio, location, website
  • avatar_url text
  • followers_count, following_count

content

  • id uuid
  • type enum: article|video|podcast|pill|course
  • author_id uuid (→ profiles)
  • slug text unique
  • status enum: draft|published
  • is_featured bool
  • cover_image_url text
  • embed_url text (video/podcast)
  • likes_count int (denormalized)
  • published_at timestamptz

content_translations

  • id uuid
  • content_id uuid (→ content)
  • locale enum: es|en
  • title text
  • excerpt text
  • body jsonb (Tiptap JSON)
  • is_auto_translated bool

categories + tags

  • categories: id, slug, name_es, name_en
  • tags: id, slug, name_es, name_en
  • content_categories: content_id, category_id
  • content_tags: content_id, tag_id

courses (extends content)

  • content_id uuid (→ content)
  • price, currency, duration
  • stripe_product_id text
  • students_count, rating

likes + bookmarks

  • user_id + content_id composite PK
  • created_at
  • — one table each, RLS: user sees only their own bookmarks

follows

  • follower_id + following_id composite PK
  • created_at

email_gates

  • email text
  • content_id uuid (→ content)
  • unlocked_at timestamptz

Key RLS Rules

  • 📖 content: anyone can read published; only author or admin can write
  • 🔒 bookmarks: user can only read/write their own rows
  • ❤️ likes: authenticated users can insert; anyone can read count
  • 👑 profiles.role: only admin can update role column
  • 🏠 homepage feed: DB view filtering role IN (admin, author) + published
